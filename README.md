### 이렇게 일합니다

**AI 에이전트를 기본값으로 씁니다.** 저장소 커밋에 `Co-Authored-By: Claude`가 그대로 남아 있습니다.
제가 하는 일은 문제를 정의하고, 작업을 자를 단위를 정하고, 나온 결과를 검토해서 받을지 정하는 것입니다.
한 번에 하나씩만 지시하는 편이라 커밋 대부분이 1~2파일 단위입니다. 몰아서 시키면 되돌릴 수가 없더군요.

**한계도 같이 배웠습니다.** 금융 계산은 그럴듯하게 틀린 값이 나옵니다.
숫자 하나가 잘못 나가면 곧 클레임이라, 결국 수기 계산으로 대조하는 QA 사이클을 따로 만들었습니다.

**실서비스에서 겪은 것들**
- 모든 요청을 정밀 모델로 돌리다 비용 리포트를 보고 → 모델 라우팅 도입
- 알림을 촘촘히 넣었더니 알림 피로로 사용자가 앱을 껐다 → "놓치면 아까운 것만" 알리도록 축소
- 프롬프트를 "될 때까지 돌리기"로 튜닝하다 사용량 소진 반복 → 규칙 기반 프롬프트 시스템으로 재설계
- API 키 잔고 소진으로 실사용자 AI가 전면 중단 → 자동 충전·키 분리로 대응
